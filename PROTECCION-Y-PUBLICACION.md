# Separacion entre version privada y version publica

## Repositorio privado

El repositorio privado conserva el trabajo estrategico y tecnico completo:

- roadmap de producto;
- requisitos internos;
- brief de patente/publicacion;
- arquitectura inicial;
- analisis juridico ampliado;
- estrategia de validacion institucional;
- decisiones sobre actores, mercado, colegios, federaciones y alianzas;
- datos de prueba y evolucion del MVP tecnico.

Este repositorio debe permanecer privado mientras se define que registrar, que presentar y que abrir publicamente.

## Repositorio publico

El repositorio publico funciona como portfolio y demo conceptual:

- muestra la interfaz;
- permite recorrer el flujo;
- explica el valor institucional;
- enseña la vertical deportiva;
- evita exponer detalles sensibles de estrategia, proteccion intelectual o negocio.

## Que proteger

- La combinacion de flujo notarial digital, cadena de custodia, evidencia, validacion profesional y repositorio verificable.
- La adaptacion a contratos deportivos federados con agentes, clubes, jugadores, sponsors, federaciones y escribanos.
- La arquitectura de roles, estados, trazabilidad, certificacion, archivo y consulta.
- La narrativa de producto como infraestructura institucional de confianza.

## Que registrar o documentar

- Nombre comercial y variantes de marca.
- Manual funcional del MVP.
- Diagramas de arquitectura.
- Flujos de usuario.
- Pantallas y material grafico.
- Documentacion tecnica y juridica propia.
- Fecha de creacion y evolucion mediante commits privados.

## Que publicar

- Demo navegable.
- README de portfolio.
- Capturas o portada.
- Explicacion de alcance y limitaciones.
- Contacto o invitacion a validar el piloto.

## Que no publicar todavia

- Estrategia de patente.
- Roadmap interno.
- Detalle completo de integraciones futuras.
- Analisis juridico profundo de documentos de terceros.
- Datos reales de personas, escribanos, agentes, federaciones o contratos.
- Claves, tokens, credenciales o endpoints productivos.
